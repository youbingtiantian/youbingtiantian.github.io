这篇是关于 vue3 的「reactive」如何实现

基于源码的结构实现，并不会一步步说明搭建项目的过程…（麻烦

reative

下面的截图是 vue 官网对于 「reactive」 的描述

由此可以知道「reactive」是返回了一个对象的响应式代理
那么我们首先要做的就是编写一个 function 叫做「reactive」

reactive.ts

首先创建了「reactive.ts」文件，同时声明了一个「reactive」函数
其接收一个当前的 target，就是大家在使用 vue 的 reacvie 时，传入进来的对象

我们在「reactive」函数中返回了一个「createReactiveObject」的函数
见名知意是创建了一个响应式对象，此函数接收当前 target 对象，同时还接收了一个 handlers 函数，和一个响应式的 weakMap 对象。

Proxy && WeakMap

大家都知道 vue3 采用 proxy 进行响应式代理，那么我们创建一个响应式对象时，就应该传递当前对象，和一个拥有 setter 和 getter 的这两个拦截方法（至少）的对象进去，第三个参数是一个 weakMap 对象，它的作用是为我们缓存之前已经被代理过的对象，以此避免一些不必要的重复创建，同时 weakMap 对象是一组键值对的集合，其中的键是「弱引用的」，这意味着在没有其他引用存在时垃圾回收能正确进行，原生 WeakMap 的结构是特殊且有效的，其用于映射的 key 只有在其没有被回收时才是有效的。

createReactiveObject

「createReactiveObject」方法接受这三个参数后，首先会从 weakMap 中取出当前被代理对象的代理对象，如果有的话则直接返回，如果当前对象并没有被创建过（代理过），那么则为其创建代理对象 「new Proxy」传入当前需要被代理的对象和拥有 getter 和 setter 方法的对象，同时将其存储到 weakMap 对象中，便于下次直接取出，在最后「return proxy」，注意，此处是返回代理对象

baseHandlers.ts

在 reactive.ts 中，我们的 mutableHandlers 是从 baseHandlers.ts 中导入的，在此文件中，我们实现了关于 getter 和 setter 方法的具体内容。

可以看到，我们在「createGetter」方法中接收了当前对象，和当前被 get 的 key，以及 receiver，我们获取了当前的 key 对应的值，同时进行依赖收集，最终返回当前数据。

然后，我们在「createSetter」方法中接收了当前对象，和当前被 set 的 key、被修改的新 value 值，以及 receiver，在其中我们对当前值进行修改，通过「Reflect.set」同时进行依赖的触发，最终返回数据

我在这里不会说明具体 api 的效果，如果不太了解的可以查询文档Reflect，Proxy

effect.ts

在此文件中我们主要实现的是模拟第一次的依赖收集 effect 方法以及收集依赖和触发依赖，主要的目的其实就是为了将依赖进行一个管理。

「track」方法是当 getter 触发时候进行依赖的收集，传入了当前被代理对象和当前 key，在其中我们使用 effect 方法进行模拟第一次触发，会创建一个「ReactiveEffect」类，将模拟第一次触发的函数传递进去进行执行，以及标记当前触发的类「 activeEffect = this 」，回到「track」方法中，我们记录了当前标记，从过往可能存储过的「targetMap」中取出当前触发对象，如果没有存储过，那么保存一个以当前对象为 key，以 Map 结构为 value 的键值对数据，然后再取出当前触发的代理对象被 getter 的 key 值对应的依赖 value，如果没有，则创建一个同样的键值对类型

「track」方法的末尾，通过「trackEffects」方法通过「dep」将当前被代理对象的 key 值触发的依赖操作进行 add 保存！

「trigger」方法是当 setter 触发时进行依赖的触发，依旧是熟悉的取值操作，将取到的值传递给「triggerEffects」后通过循环的方式触发之前存储的「ReactiveEffect」类中标记了的「 activeEffect = this 」的 run 方法，从而达到触发依赖达到值修改后，对应的依赖也进行执行

dep.ts

在 effect.ts 文件中定义的 type Dep 是 Set 结构的 ReactiveEffect 数据

同时创建 dep 数据，可能有多个依赖所以也是 Set 结构的数据（避免重复）
